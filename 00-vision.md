# 00 · Vision · 北极星与一句话定位

## 一句话定位

> **Sentinel 是让普通人通过自然对话做出真实可上线、可演化、可运营的全栈应用的 AI 合伙人。**

## 长格式定位

我们让任何不会编程的人，能像和合伙人聊天一样：

1. **说出**他想做的应用 → 5 分钟拿到可分享 URL
2. **持续告诉**它「再加个功能」「这里改改」 → 应用持续演化而不崩溃
3. **看到上线后的数据**，对话式优化 → 真实运营，挣到真实钱

## 我们解决的真实问题

> 「我有个想法，但我不会写代码 / 找不到合适的开发者 / 没钱请团队。」

这是一个长期未被很好解决的问题：

| 现有方案 | 真实痛点 |
|---|---|
| 找外包 | 贵、沟通成本高、改一次再花一次钱 |
| 用 SaaS（Wix / Webflow / 有赞） | 受平台规则约束、迁移困难、个性化低 |
| 学编程 | 学习曲线 6 个月起，绝大多数人放弃 |
| Bolt / Lovable / v0 | 5 分钟出 demo，但**做完就没下文**：改不动、上不了线、看不到数据 |

Sentinel 的位置 —— **不止帮你做出来，还陪你做下去。**

## 核心信条

### 信条 1：交付的是产品，不是代码

绝大多数普通人不关心代码长什么样。Sentinel 默认隐藏所有代码，**只暴露应用本身**（预览 URL + 数据看板 + 对话框）。代码是实现细节，不是产品。

### 信条 2：应用必须真实可运行、真实可分享

「demo」是没用的。一个应用如果不能让用户拿到真实 URL、用真实域名、给真实用户用，它对普通人就毫无价值。

**所有 Sentinel 生成的应用必须是 production-ready 的，第一秒就能给真人用。**

### 信条 3：演化优先于生成

一次性生成不是难事，难的是**第 5 次、第 50 次改动后还能跑、还合理、还能继续改**。

Sentinel 的核心壁垒是**让应用越长越大而不是越改越烂**。

### 信条 4：上线只是开始

Bolt 的故事在「点击导出到 Netlify」处结束。Sentinel 的故事从那里**开始**：

- 上线后看到数据
- 收到用户反馈
- 对话式优化
- A/B 验证
- 持续迭代

这是普通人**做产品**而不是**做 demo** 的真实诉求。

### 信条 5：结果偏差是系统责任

当 AI 做出的东西偏离用户期望，**默认是 Sentinel 的责任**，不甩锅给「用户提示词不好」「底模不行」。每个偏差都要回归到：

- 意图归一化是否到位
- 澄清是否充分
- 沙箱里是否真跑过
- 部署是否真成功
- 用户是否真满意

并通过改进 Sentinel 的能力来修复。

## 5 年愿景

到 2031 年，**至少 100 万普通人**通过 Sentinel 做出了**仍在运营**的真实应用，其中：

- 至少 1 万个应用月活跃用户 > 100
- 至少 1000 个应用通过应用本身产生月收入 > 1 万元
- 至少 100 个应用月收入 > 10 万元

**衡量标准不是"用过 Sentinel"，而是"靠 Sentinel 持续做出有用的东西"。**

---

## 不是什么（防止漂移到错位）

- **不是 IDE**（不和 Cursor / Windsurf / Trae / Kiro 竞争）
- **不是代码生成器**（不和 GitHub Copilot / Codeium 竞争）
- **不是 demo 工厂**（区别于 v0 / Bolt 的核心定位）
- **不是 SaaS 模板平台**（区别于 Wix / Webflow）
- **不是企业内部工具构建器**（区别于 Retool / Tooljet，至少 Phase 1-3 不做）

详见 `14-non-goals.md`。

## 北极星指标（NSM）

**Weekly Active Operating Apps**：每周有真实用户访问 + 当周收到至少一次用户对话改动的应用数。

这个指标同时衡量：

- 应用真实在用（有访问）
- 创作者持续在迭代（有对话）
- 不是一次性 demo

短期次级指标（前 12 个月）：

- 注册 → 第一次部署成功率（目标 ≥ 70%）
- 第一次部署 → 第二次有效演化的转化率（目标 ≥ 40%）
- D30 留存（目标 ≥ 25%）
