# 10 · Product Overview · 产品全景

## 产品三大模块

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│   Create        →     Evolve         →     Operate      │
│   做出来              改下去                跑起来        │
│                                                         │
│  • 对话生成      • 持续演化           • 真实部署         │
│  • 澄清意图      • 影响面分析         • 数据看板         │
│  • 即时预览      • 自动测试           • 反馈对话         │
│  • 一键部署      • 不破坏既有功能     • A/B 实验         │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

每个模块都是**完整可用的**，但合在一起才是 Sentinel 的核心价值（市场上其他产品只有 Create）。

---

## 模块 1：Create · 5 分钟从想法到上线

### 用户感知到的步骤

1. 进入站点 → 输入「我想做一个 X」
2. AI 追问 3-5 个澄清问题（澄清完整度可视化）
3. AI 生成应用 → 实时预览（左对话 / 右预览）
4. 点击「上线」→ 30 秒后拿到 yourapp.sentinel.app
5. 点击「分享」→ 转发链接给朋友

### 后台真实发生的

- IntentNormalize → spec
- HumanGate（澄清不充分时阻断）
- TaskBreakdown → 子任务
- Worker 分块生成代码到 Sandbox
- 自动 e2e smoke test
- 部署到 Cloudflare Pages + 自动配 DB
- 数据回写 IntentGraph + EpisodicMemory

### 时间承诺

- **首次进入到看到第一个像样应用预览**：≤ 3 分钟
- **首次部署上线**：≤ 5 分钟（含选域名）
- **第 2 次以后的部署**：≤ 30 秒

## 模块 2：Evolve · 持续演化不崩坏

### 用户感知到的步骤

1. 在已有应用页打开对话框
2. 说「再加一个 X 功能」/「这里改成 Y」
3. AI 提示「这会影响 [A][B][C]，确定吗？」
4. 确认后 → 增量改动 → 自动测试 → 部署

### 后台真实发生的

- 解析意图 → 更新 IntentGraph 节点
- CodeGraph 影响面分析 → 列出关联模块
- 检索 EpisodicMemory 找到类似演化的成功 / 失败经验
- FailureTaxonomy 提前规避已知风险
- 增量改动到 Sandbox
- 自动跑既有 e2e + 新增功能 e2e
- 通过 → 部署
- 失败 → 回滚 + 报告原因

### 关键能力承诺

- 第 1-10 次演化：100% 不破坏既有功能
- 第 10-50 次演化：≥ 90% 不破坏
- 第 50+ 次演化：≥ 80% 不破坏（行业基线 < 30%）

---

## 模块 3：Operate · 上线后的运营对话

### 用户感知到的步骤

1. 应用上线后，应用页出现「运营」tab
2. 看到访问量 / 用户停留 / 转化漏斗 / 错误日志
3. AI 主动提示：「过去 7 天注册→付款转化率 12%，行业基线 18%，要不要看看为什么？」
4. 用户：「好，看一下」
5. AI：「我看到 76% 用户在选套餐时离开，可能是套餐说明不够清晰，要不要试试简化套餐展示？我准备了 A/B 方案」
6. 用户：「试试」→ A/B 上线 → 7 天后回看结果

### 后台真实发生的

- 应用模板自带轻量 Telemetry SDK（PostHog 协议）
- 部署时自动注入项目 ID
- 数据回流到 Sentinel 数据看板
- LLM 周期性扫数据找异常 / 机会
- 用户授权后才主动提示

### 关键差异化

**这是 Bolt / Lovable / v0 / Replit 都没有的能力。** 是真正的护城河。

## 三大模块的产品边界

| 模块 | 输入 | 输出 | 关键 SLA |
|---|---|---|---|
| Create | 自然语言意图 | 可访问的应用 URL | 5 分钟 / 部署成功率 ≥ 70% |
| Evolve | 改动意图 | 增量更新的应用 | 30 秒 / 不破坏率 ≥ 90% |
| Operate | 数据 + 反馈 | 优化对话 + 实验 | 实时 / 主动提示 ≥ 1 次/周 |

## 产品形态

### 主形态：Web App（响应式）

- 桌面浏览器（主）
- 移动端浏览器（次，Phase 2 完善）

### 不做的形态

- ❌ 桌面客户端（vscode fork 已废弃）
- ❌ 移动端原生 App（Phase 1-3）
- ❌ CLI / SDK（Phase 1-3，等后期再考虑给开发者用）
- ❌ 浏览器扩展（无需要）

---

## 用户进入路径

### 入口 1：landing 页 → 直接对话生成（主路径）

未登录也能体验前 2-3 轮对话，部署时引导注册。

### 入口 2：示例画廊 → fork 已有应用

新手不知道做什么时，可以从 10 个内置示例中选一个改。

### 入口 3：分享链接 → 看到别人做的应用 → "我也做一个"

社交传播的核心入口，链接末尾自带「Make your own」CTA。

---

## 用户离开后还能干什么

- 应用持续在线（Cloudflare Pages 部署，不依赖 Sentinel 服务器）
- 用户可下载完整代码（Phase 2 提供）
- 用户可绑定自己域名
- 用户可不再用 Sentinel，但应用仍能跑

**这是产品自信的体现：不锁用户，靠价值留住。**
